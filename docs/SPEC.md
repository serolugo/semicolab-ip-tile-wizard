## 1. Scope

This document specifies the behavior of TileWizard V1, a CLI tool that converts generic Verilog RTL into tiles compatible with the SemiCoLab multi-project ASIC platform. It defines the fixed tile interface, command behavior, input format, generated output format, and all validation rules including fatal errors and non-blocking warnings.

---

## 2. SemiCoLab tile interface

All tiles generated by TileWizard expose exactly the following 9 ports. This interface is fixed and cannot be modified by the user.

| Port | Direction | Width | Type |
|------|-----------|-------|------|
| `clk` | input | 1 | wire |
| `arst_n` | input | 1 | wire |
| `csr_in` | input | 16 | wire |
| `data_reg_a` | input | 32 | wire |
| `data_reg_b` | input | 32 | wire |
| `data_reg_c` | output | 32 | wire |
| `csr_out` | output | 16 | wire |
| `csr_in_re` | output | 1 | wire |
| `csr_out_we` | output | 1 | wire |

### 2.1 csr_in bit zones

| Bits | Type | Description |
|------|------|-------------|
| `[15:12]` | Single pulse | Asserted for one clock cycle by the Python interface, then automatically cleared. |
| `[11:4]` | Stable | Hold their value until explicitly changed by the Python interface. |
| `[3:0]` | Clear on read | Cleared by the Python interface when the tile reads the input data correctly. |

### 2.2 csr_out bit zones

| Bits | Type | Description |
|------|------|-------------|
| `[15:4]` | Stable | Hold their value. Read periodically by the Python interface. |
| `[3:0]` | Clear on read | Cleared by the Python interface upon reading. Used as event indicators. |

## 3. Command specifications

### 3.1 `tilewizard init <project_name>`

**Preconditions:** `<project_name>` must not exist in the current directory.

**Effects:**
- Creates `<project_name>/` directory.
- Creates `<project_name>/src/` directory.
- Creates `<project_name>/src/.gitkeep` file.
- Prints next-step instructions to stdout.

**Fatal condition:** `<project_name>` already exists â†’ exits with code 1.

---

### 3.2 `tilewizard parse --top <top_module>`

**Preconditions:**
- Must be run from inside a project directory (i.e., `src/` must exist in CWD).
- `src/<top_module>.v` must exist.
- The file must contain a module declaration matching `module <top_module> (...)`.

**Effects:**
- Parses port names from the module header using regex.
- Generates `ip_config.yaml` in CWD with all detected ports listed under `ports:`.
- Prints the list of detected ports to stdout.

**Port parsing rules:**
- Handles ANSI-style port declarations: `input wire [N:0] port_name`
- Handles declarations without net type: `input port_name`
- Handles `wire`, `reg`, `logic` net types.
- Extracts port name only â€” direction and width are not stored in the YAML.
- Duplicate port names are discarded.

**Fatal conditions:**
- `src/` not found in CWD.
- `src/<top_module>.v` not found.
- Module declaration for `<top_module>` not found in the file.
- No ports detected in the module.

---

### 3.3 `tilewizard wrap`

**Preconditions:**
- Must be run from inside a project directory containing `ip_config.yaml`.
- All required fields in `ip_config.yaml` must be non-empty.
- `src/` must contain at least one `.v` file.
- `top_module` must be declared as a module in one of the `.v` files in `src/`.

**Effects:**
- Validates `ip_config.yaml` (see Section 7).
- Validates port mapping (see Section 5 and Section 7).
- Emits non-blocking warnings to stdout.
- Creates output directory tree under `<ip_tile_name>/`.
- Copies all `.v` files from `src/` to both output trees.
- Generates all output files (see Section 6).
- Prints a summary to stdout.

**Exit:** Code 0 on success (warnings may be present). Code 1 on any fatal error.

## 4. ip_config.yaml specification

### 4.1 Required fields

| Field | Type | Description |
|-------|------|-------------|
| `ip_tile_name` | string | Name of the tile and output folder. Used as the module name prefix for the wrapper. |
| `top_module` | string | Exact Verilog module name of the IP to wrap. |
| `description` | string | Non-empty human-readable description of the IP. |
| `author` | string | Non-empty author name. |

### 4.2 Optional fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `version` | string | `"1.0.0"` | Version string included in generated file headers. |
| `ports` | map | `{}` | Port mapping. Key: IP port name. Value: tile port expression. |

### 4.3 Port mapping value syntax

```
<tile_port>
<tile_port>[<bit>]
<tile_port>[<hi>:<lo>]
```

Where `<tile_port>` is one of the 9 SemiCoLab ports. A null or empty value is valid and generates warning `TW-W01`.

## 5. Port mapping rules

### 5.1 Valid mapping targets

Only the 9 ports defined in Section 2 are valid mapping targets. Any other name triggers `TW-E08`.

### 5.2 Slice validity

For bus ports (`csr_in`, `data_reg_a`, `data_reg_b`, `data_reg_c`, `csr_out`), slices must satisfy:

- `hi â‰Ą lo â‰Ą 0`
- `hi < port_width`

For scalar ports (`clk`, `arst_n`, `csr_in_re`, `csr_out_we`), no slice or only `[0:0]` / `[0]` is accepted.

### 5.3 Bit conflict detection

Each bit of each tile port may be assigned to at most one IP port. Mapping two IP ports to the same tile bit triggers `TW-E06`.

### 5.4 Unassigned output bits

Bits of `data_reg_c` or `csr_out` not covered by any mapping are automatically driven to zero with an `assign` statement. Warning `TW-W02` is emitted for each unassigned range.

### 5.5 Unmapped control outputs

- `csr_out_we` not mapped â†’ `assign csr_out_we = 1'b1;` + `TW-W03`
- `csr_in_re` not mapped â†’ `assign csr_in_re = 1'b0;` + `TW-W04`
- `data_reg_b` not mapped â†’ no assignment, `TW-W05` only

## 6. Generated files specification

### 6.1 `<ip_tile_name>_wrapper.v`

A synthesizable Verilog 2001 wrapper module. Structure:

```
`timescale 1ns / 1ps

module <ip_tile_name>_wrapper #(
    parameter REG_WIDTH     = 32,
    parameter CSR_IN_WIDTH  = 16,
    parameter CSR_OUT_WIDTH = 16
)( <9 SemiCoLab ports> );

    <top_module> u_<top_module> (
        .<ip_port>(<tile_signal>),
        ...
    );

    // assign statements for unmapped output bits and control ports

endmodule
```

Generated with Jinja2. The file is identical in `output/rtl/` and `veriflow/src/rtl/`.

### 6.2 `tile_config.yaml`

```yaml
tile_name: "<ip_tile_name>"
tile_author: "<author>"
top_module: "<ip_tile_name>_wrapper"
description: |
  <description>
ports: |
  <tile_port>[slice] â†’ <ip_port>   (one line per mapped port)
usage_guide: |
  # TODO
tb_description: |
  # TODO
```

### 6.3 `run_config.yaml`

```yaml
run_author: ""
objective: ""
tags: ""
main_change: |
notes: |
```

All fields empty â€” to be filled by the user before each VeriFlow run.

### 6.4 `tb_tile.v`

Fixed-structure testbench compatible with VeriFlow. Required markers:

- `/* MODULE_INSTANTIATION */` â€” VeriFlow replaces this with the DUT instantiation.
- `/* USER_TEST */` â€” marks the beginning of the user test section.
- `// USER TEST STARTS HERE //` and `// USER TEST ENDS HERE //` â€” delimit the editable area.

Pre-stimulus lines generated between the markers follow these rules:

| Mapping target | Generated line |
|----------------|----------------|
| `data_reg_a` | `write_data_reg_a(32'h00000000); // <ip_port> (IP input)` |
| `data_reg_b` | `write_data_reg_b(32'h00000000); // <ip_port> (IP input)` |
| `csr_in` / slice | `write_csr_in(16'h0000); // <ip_port> â†’ csr_in[slice]` |
| `data_reg_c` | `$display("<ip_port> = %0h", data_reg_c);` |
| `csr_out` / slice | `$display("<ip_port> = %0b", csr_out[slice]);` |
| `data_reg_b` unmapped | `// write_data_reg_b(32'h00000000); // sin mapeo` |

A `@(posedge clk);` separator is inserted between write and read stimulus lines.

### 6.5 `output/docs/README.md`

Tile documentation in Markdown. Contains: metadata table, full SemiCoLab interface table, `csr_in` and `csr_out` bit-zone detail tables, and a `## Uso` section for the user to fill in.

### 6.6 `output/docs/port_map.md`

Markdown table with columns: IP Port, Tile Port, Signal expression. One row per port in `ip_config.yaml`, including unmapped ports.

## 7. Validation rules

### 7.1 Fatal errors

All fatal errors print `[TW-Exx] <message>` to stdout and exit with code 1. No output files are generated after a fatal error.

| Code | Condition |
|------|-----------|
| `TW-E01` | `ip_config.yaml` not found in CWD |
| `TW-E02` | `ip_config.yaml` contains invalid YAML syntax |
| `TW-E03` | Any of `ip_tile_name`, `top_module`, `description`, `author` is absent or empty |
| `TW-E04` | `src/` not found or contains no `.v` files |
| `TW-E05` | No `.v` file in `src/` contains `module <top_module>` |
| `TW-E06` | Two IP ports mapped to the same bit of the same tile port |
| `TW-E07` | Slice syntax invalid or out of range for the target tile port |
| `TW-E08` | Mapping target is not a valid SemiCoLab port name |

### 7.2 Non-blocking warnings

Warnings are printed before generation begins. They do not prevent output from being generated.

| Code | Condition | Automatic remediation |
|------|-----------|----------------------|
| `TW-W01` | IP port with null or empty mapping value | Port omitted from wrapper instantiation |
| `TW-W02` | One or more bits of `data_reg_c` or `csr_out` not covered | `assign <port>[hi:lo] = N'b0;` |
| `TW-W03` | `csr_out_we` not mapped to any IP port | `assign csr_out_we = 1'b1;` |
| `TW-W04` | `csr_in_re` not mapped to any IP port | `assign csr_in_re = 1'b0;` |
| `TW-W05` | `data_reg_b` not mapped to any IP port | No assignment generated |
