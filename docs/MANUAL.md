## 1. Overview

SemiCoLab tiles are Verilog modules that expose a fixed 9-port interface so they can be integrated into the SemiCoLab multi-project ASIC platform. TileWizard takes any existing IP in Verilog and produces a complete tile — wrapper RTL, testbench scaffold, and VeriFlow configuration files — through a three-command workflow.

TileWizard does not modify the original IP. It generates a wrapper module that instantiates the IP and connects its ports to the SemiCoLab interface according to a mapping defined by the user in `ip_config.yaml`.

## 2. Installation

**Requirements:** Python 3.10+, PyYAML ≥ 6.0, Jinja2 ≥ 3.1.

```bash
pip install -r requirements.txt
pip install -e .        # development — editable install
# or
pip install .           # production / CI
```

After installation the `tilewizard` command is available globally in the active Python environment.

## 3. Workflow

```
tilewizard init <project_name>
        │
        ▼
   Place .v files in src/
        │
        ▼
tilewizard parse --top <top_module>
        │
        ▼
   Edit ip_config.yaml
   (fill description, author, port mapping)
        │
        ▼
tilewizard wrap
        │
        ▼
   <ip_tile_name>/output/      ← tile deliverable
   <ip_tile_name>/veriflow/    ← ready for VeriFlow
```

## 4. Command reference

### `tilewizard init <project_name>`

Creates the project folder structure.

```bash
tilewizard init my_ip
```

**Creates:**
```
my_ip/
└── src/
    └── .gitkeep
```

**Errors:**
- Exits with error if `<project_name>` already exists in the current directory.

---

### `tilewizard parse --top <top_module>`

Scans `src/<top_module>.v`, extracts port names using regex, and generates `ip_config.yaml` in the current directory.

```bash
cd my_ip
tilewizard parse --top my_ip
```

**Requires:**
- `src/` folder in the current directory
- `src/<top_module>.v` exists
- Module declaration `module <top_module> (...)` present in the file

**Generates:** `ip_config.yaml` with all detected ports listed under `ports:`, ready to be filled in.

**Errors:**
- `src/` not found
- `<top_module>.v` not found in `src/`
- Module declaration not found in the file

---

### `tilewizard wrap`

Reads `ip_config.yaml`, validates all fields and the port mapping, and generates the complete tile output.

```bash
tilewizard wrap
```

**Requires:** `ip_config.yaml` in the current directory with all required fields filled in.

**Generates:** Full `<ip_tile_name>/` output tree (see [Section 7](#7-output-files-reference)).

**Non-blocking warnings** are printed to stdout before the summary. Fatal errors exit with code 1.

## 5. ip_config.yaml reference

Generated by `tilewizard parse`. Must be edited before running `tilewizard wrap`.

```yaml
ip_tile_name: my_ip        # name for the generated tile and output folder
top_module:   my_ip        # Verilog module name to instantiate
description:  "..."        # required — describe what the IP does
author:       "..."        # required — your name
version:      "1.0.0"      # tile version string

ports:
  <ip_port>: <tile_port>[slice]
  ...
```

**Required fields:** `ip_tile_name`, `top_module`, `description`, `author`.

**Port mapping syntax:**

| Example | Meaning |
|---------|---------|
| `clk: clk` | Connect IP port `clk` to tile port `clk` |
| `data_in: data_reg_a` | Full 32-bit bus |
| `ctrl: csr_in[3:0]` | 4-bit slice of `csr_in` |
| `flag: csr_in[4]` | Single bit |
| `result: data_reg_c` | Full 32-bit output bus |

Leaving a port value empty or null generates warning `TW-W01` and the port is left unconnected.

## 6. Port mapping guide

### SemiCoLab tile interface

| Port | Dir | Width |
|------|-----|-------|
| `clk` | input | 1 |
| `arst_n` | input | 1 |
| `csr_in` | input | 16 |
| `data_reg_a` | input | 32 |
| `data_reg_b` | input | 32 |
| `data_reg_c` | output | 32 |
| `csr_out` | output | 16 |
| `csr_in_re` | output | 1 |
| `csr_out_we` | output | 1 |

### csr_in bit zones

`csr_in` is driven by the Python interface. Its 16 bits are divided into three zones with different behaviors:

| Bits | Type | Behavior |
|------|------|----------|
| `[15:12]` | **Single pulse** | Asserted for one clock cycle, cleared automatically. Use for commands or instructions triggered from Python. |
| `[11:4]` | **Stable** | Hold their value until Python changes them. Use for configuration, mode selection, or parameters. |
| `[3:0]` | **Clear on read** | Cleared when the circuit reads the input correctly. Use for orders that confirm their own reception. |

### csr_out bit zones

`csr_out` is read by the Python interface:

| Bits | Type | Behavior |
|------|------|----------|
| `[15:4]` | **Stable** | Hold their value. Use for status flags or output values that Python reads periodically. |
| `[3:0]` | **Clear on read** | Cleared when Python reads the register. Use for event indicators or single-read flags. |

### Mapping strategy

- **1-bit input** with no timing constraint → `csr_in[N]` in stable zone `[11:4]`
- **1-bit input** that acts as a trigger → `csr_in[N]` in single-pulse zone `[15:12]`
- **1-bit input** that confirms a handshake → `csr_in[N]` in clear-on-read zone `[3:0]`
- **32-bit input** → `data_reg_a` or `data_reg_b`
- **32-bit output** → `data_reg_c`
- **1-bit output** status flag → `csr_out[N]` in stable zone `[15:4]`
- **1-bit output** event/done pulse → `csr_out[N]` in clear-on-read zone `[3:0]`

### Unassigned output bits

Any output bits not covered by the mapping are automatically driven to zero by TileWizard with a `TW-W02` warning. This is safe and expected — not all bits need to be used.

## 7. Output files reference

```
<ip_tile_name>/
├── output/
│   ├── rtl/
│   │   ├── <original .v files copied from src/>
│   │   └── <ip_tile_name>_wrapper.v
│   └── docs/
│       ├── README.md
│       └── port_map.md
└── veriflow/
    ├── tile_config.yaml
    ├── run_config.yaml
    └── src/
        ├── rtl/
        │   ├── <original .v files copied from src/>
        │   └── <ip_tile_name>_wrapper.v
        └── tb/
            └── tb_tile.v
```

### `<ip_tile_name>_wrapper.v`

The SemiCoLab-compatible wrapper. Instantiates the original IP module and connects its ports according to `ip_config.yaml`. Unassigned output bits and unmapped control ports receive default `assign` statements.

### `tile_config.yaml`

VeriFlow tile configuration. Contains tile metadata and the complete port mapping (SemiCoLab interface → IP port). Edit `usage_guide` and `tb_description` before running VeriFlow.

### `run_config.yaml`

VeriFlow run configuration. Fill in `run_author`, `objective`, `tags`, and `main_change` before each verification run.

### `tb_tile.v`

Testbench scaffold for VeriFlow. Declares all tile signals, includes the `MODULE_INSTANTIATION` and `USER_TEST` markers, and pre-populates stimulus calls based on the port mapping:

- Mapped to `data_reg_a` → `write_data_reg_a(32'h00000000);`
- Mapped to `data_reg_b` → `write_data_reg_b(32'h00000000);`
- Mapped to `csr_in` or slice → `write_csr_in(16'h0000);`
- Mapped to `data_reg_c` → `$display("<port> = %0h", data_reg_c);`
- Mapped to `csr_out` or slice → `$display("<port> = %0b", csr_out[N:M]);`

### `output/docs/README.md`

Tile documentation with metadata table and full SemiCoLab interface description including bit-zone details.

### `output/docs/port_map.md`

Markdown table of the complete IP-to-tile port mapping.

## 8. Error and warning reference

### Fatal errors — exit code 1

| Code | Trigger | Action |
|------|---------|--------|
| `TW-E01` | `ip_config.yaml` not found | Place the file in the current directory |
| `TW-E02` | Invalid YAML syntax | Fix the YAML formatting in `ip_config.yaml` |
| `TW-E03` | Required field empty or missing | Fill in `ip_tile_name`, `top_module`, `description`, `author` |
| `TW-E04` | `src/` missing or no `.v` files | Place at least one `.v` file in `src/` |
| `TW-E05` | `top_module` not declared in any `.v` | Check the module name in `ip_config.yaml` matches the Verilog source |
| `TW-E06` | Two IP ports mapped to the same tile bit | Each bit of each tile port can only be assigned once |
| `TW-E07` | Slice out of range for the tile port width | Check bit indices against the port widths in Section 6 |
| `TW-E08` | Tile port name not in SemiCoLab interface | Only the 9 ports listed in Section 6 are valid targets |

### Non-blocking warnings

| Code | Trigger | Automatic action |
|------|---------|-----------------|
| `TW-W01` | IP port with no `map_to` value | Port ignored, commented in testbench |
| `TW-W02` | Unassigned bits on an output bus | `assign <port>[N:M] = 0;` generated |
| `TW-W03` | `csr_out_we` not mapped | `assign csr_out_we = 1'b1;` generated |
| `TW-W04` | `csr_in_re` not mapped | `assign csr_in_re = 1'b0;` generated |
| `TW-W05` | `data_reg_b` not mapped | No assignment, documented in summary |

## 9. Examples

### counter — minimal mapping

A 32-bit up-counter with synchronous load. Exercises basic port mapping including a single-bit `csr_in` slice.

`ip_config.yaml`:
```yaml
ip_tile_name: counter
top_module:   counter
description:  "32-bit up-counter with synchronous load and overflow flag."
author:       "Your Name"
version:      "1.0.0"

ports:
  clk:      clk
  rst_n:    arst_n
  load_val: data_reg_a
  load_en:  csr_in[0]
  count:    data_reg_c
  overflow: csr_out[0]
```

### sat_alu — full mapping

A saturating ALU. Uses both data registers, multiple `csr_in` zones, and multiple `csr_out` flags.

`ip_config.yaml`:
```yaml
ip_tile_name: sat_alu
top_module:   sat_alu
description:  "32-bit saturating ALU with 8 operations and status flags."
author:       "Your Name"
version:      "1.0.0"

ports:
  clk:         clk
  rst_n:       arst_n
  operand_a:   data_reg_a
  operand_b:   data_reg_b
  op_mode:     csr_in[11:8]
  shift_amt:   csr_in[6:2]
  start:       csr_in[15]
  clear_flags: csr_in[0]
  result:      data_reg_c
  overflow:    csr_out[4]
  zero:        csr_out[5]
  negative:    csr_out[6]
  done:        csr_out[0]
```

Expected warnings: `TW-W03`, `TW-W04` (control pulses not driven by IP), `TW-W02` for unused `csr_out` and `csr_in` bits.
